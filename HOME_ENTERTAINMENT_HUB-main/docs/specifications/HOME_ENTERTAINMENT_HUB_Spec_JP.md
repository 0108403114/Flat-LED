# HOME_ENTERTAINMENT_HUB 仕様書

## 1. 使用シーン

### Scene 1: ライブ空間化

- YouTube や U-NEXT のライブ映像を視聴する。
- 中央ノードは部屋の音をマイクで取得し、音量、ピーク、エネルギー変化を抽出する。
- 抽出した特徴量から、色、明るさ、変化速度、エフェクト種別を決めて照明ノードへ配信する。
- 目的は、映像視聴体験をリビング全体へ拡張すること。

### Scene 2: ヒーリング空間化

- 就寝前にスマホまたは PC で音楽を再生する。
- スマホ再生時はマイク入力、PC 再生時は AudioLINKCORE の制御信号入力を利用する。
- 演出は低刺激で、暖色寄り、ゆっくりとしたフェードを重視する。
- 目的は、睡眠前の没入感とリラックス感を高めること。

## 2. システム構成

### 中央ノード

- マイク入力を解析する。
- AudioLINKCORE から USB またはシリアルで制御信号を受信する。
- シーン切替と演出パラメータ生成を行う。
- 照明ノードへ Wi-Fi で演出パラメータを配信する。

### 照明ノード

- Wi-Fi 経由で中央ノードの演出パラメータを受信する。
- ローカル LED を描画する。
- 必要に応じて状態やエラーを中央ノードへ返す。

## 3. 必須機能

- マイク入力による音特徴量抽出
- 手動シーン切替
- Wi-Fi による複数ノード同期
- ライブ向け演出プリセット
- ヒーリング向け演出プリセット
- AudioLINKCORE 外部制御入力
- ノード状態監視

## 4. 初期版の範囲外

- HDMI 直接音声取得
- TV との Bluetooth 音声受信
- クラウド連携
- 基板量産設計

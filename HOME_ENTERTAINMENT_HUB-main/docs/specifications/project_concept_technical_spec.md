## 1. 目的

HOME_ENTERTAINMENT_HUB は、リビング空間を音と光の演出で変化させる分散型ライティングシステムである。

このプロジェクトの主目的は、中央ノードが音や外部制御を解析し、複数の LED ノードへ演出制御情報を配信することで、空間全体の雰囲気を同期的に変化させることである。

中央ノードとローカル LED ノードは、リビングテーブル上に常設するテーブルランタンのような存在として想定する。PC を立ち上げなくても単体で起動し、普段の生活空間に自然に置いておけることを重要な設計条件とする。

## 2. 想定ユースケース

1. リビングテーブル上の中央ノード + ローカル LED ノードを常設し、日常時はランタンのように空間へなじませる
2. テレビをつけて音楽が鳴り始めたら、周囲の LED と連動してリビング全体をライブ空間へ早変わりさせる
3. 就寝前はお気に入りの音楽に合わせて、低刺激で没入感のあるヒーリング空間を作り、睡眠前の時間を整える

## 3. システム全体像

### 中央ノード

- 音入力または外部制御入力を受ける
- シーンを決定する
- プリセット情報を LED ノードへ配信する
- ローカル LED ノードへは UART で送る
- サテライト LED ノードへは Wi-Fi で送る
- 家庭内の常設機器として単体起動と簡単操作を成立させる

### ローカル LED ノード

- 中央ノードに UART 直結する
- microSD からアセットを読む
- ESP32-S3 上で最終フレームを合成する
- WS2812 系 LED を駆動する

### サテライト LED ノード

- 中央ノードから Wi-Fi で制御を受ける
- 各ノードに固定 ID を持つ
- microSD からアセットを読む
- ESP32-S3 上で最終フレームを合成する
- WS2812 系 LED を駆動する

## 4. ハードウェア構成方針

### 中央ノード候補

- MCU: Seeed Studio XIAO ESP32S3 Plus を第一候補
- マイク: INMP441
- 表示: GMT020-01 TFT
- 入力: タクトスイッチ
- 外部制御: AudioLINKCORE から USB または UART で入力

### LED ノード候補

- MCU: ESP32-S3 系
- ストレージ: microSD
- LED: WS2812 系
- 出力: 最終目標は 4 から 6 ライン並列
- MVP は 1 ライン少数 LED から開始

## 5. 電源設計方針

- WS2812 系 LED の 60 mA / LED は理論最大参考値としてのみ扱う
- 本プロジェクトの通常運用では各 LED マトリクスごとに約 2.5 A 以内へ制限する
- 各 LED マトリクス用電源は 5 V / 3 A 級を基本候補とする
- 中央ノード電源と LED ノード電源は分離を基本とする
- 理論最大電流と通常運用電流は分けて管理する

## 6. ソフトウェア構成方針

### 中央ノードの責務

- マイク入力解析
- 外部制御受信
- シーン判定
- プリセット生成
- UART 配信
- Wi-Fi 配信
- UI 表示
- 家庭用常設機器としての起動直後の自律動作
- テーブル上で扱いやすい最小限のローカル操作

### LED ノードの責務

- コマンド受信
- アセット読み込み
- フレーム合成
- LED 出力
- 安全停止処理

## 7. 通信仕様の初期方針

### 物理トポロジ

- 中央ノード -> ローカル LED ノード: UART
- 中央ノード -> サテライト LED ノード: Wi-Fi

### 中央ノードから送る情報

1. メインアニメーションプリセット番号
2. エフェクトプリセット番号
3. カラーパレットプリセット番号
4. テキスト情報
5. 強制リセット

### 初期仕様で固定済みの要点

- サテライトノード ID は 1 から 16 程度の短い数値
- 個別制御は標準 3 秒で全体同期へ戻る
- ACK は UART のみ
- Wi-Fi は基本 ACK なし
- 重要コマンドは 50 ms 間隔で 3 回送る
- group_id は将来拡張用に予約する
- テキストは表示にも内部ラベルにも使う
- テキスト表示初期状態はローカル LED ノードだけ有効
- エフェクトは簡易ブレンドで重ねる

## 8. ストレージ方針

- 大容量演出データは各 LED ノードの microSD に置く
- 中央ノードは大容量フレームを送らず、制御情報のみ送る
- LED ノードは受信したプリセット番号をキーにローカルアセットを読む
- MVP では単純な生 RGB に近い形式から始める

## 9. MVP 方針

### MVP の目的

- 完成を目指すのではなく、破綻点とボトルネックを発見する
- 中央ノード性能、UART 連携、microSD + WS2812 並立、電源安定性を確認する
- 家庭用常設機器として違和感なく使える最小構成を見極める

### MVP の範囲

- 中央ノード 1 台
- ローカル LED ノード 1 台
- UART 通信
- microSD 読み出し
- 1 ライン少数 LED 出力

### MVP で後回しにするもの

- 多ノード Wi-Fi 同期
- 複雑なシーン切替
- AudioLINKCORE 本統合
- 4 から 6 ライン並列の最終形

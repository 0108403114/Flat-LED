# ハード仕様書（構想と現状）

最終更新: 2026-05-06

## 1. ハードウェアの全体像

HOME_ENTERTAINMENT_HUB は、中央ノードと複数の LED ノードで構成する分散型システムです。

- 中央ノード: 入力・解析・UI・全体制御を担当
- サテライト LED ノード: 背景とテキストをローカル合成して LED を駆動
- 将来構成: 中央ノード 1 台 + サテライト多数
- 現在の実機構成: サテライト 2 台

## 2. 現在の実機構成

### サテライトノード

- MCU: Waveshare ESP32-S3-Zero
- 台数: 2 台
- Node 1: Zone 1 / Node 1 / COM3 / MAC a0:f2:62:f0:84:90
- Node 2: Zone 1 / Node 2 / COM4 / MAC 3c:0f:02:e1:a8:20
- LED 出力:
  - Node 1 は GPIO12
  - Node 2 は GPIO13
- LED 数: 各ノード 440、合計 880
- 表示面: 88 x 10 の全体マトリクスとして運用

### 表示方式

- 1 台あたり 88 x 5 のストリップ集合を担当
- 2 台で上下または左右に連結した 88 x 10 相当として扱う
- 配線は sawtooth / serpentine 前提
- テキスト描画は 5x8 ASCII フォントを利用

## 3. 中央ノードの完成形構想

中央ノードは現時点では構想中心です。

- MCU 候補: ESP32-S3 系
- 入力:
  - INMP441 マイク
  - AudioLINKCORE 外部制御入力
  - ボタン
- 表示: GMT020-01 TFT
- センサ:
  - BH1750
  - INA228 x 2
- 役割:
  - シーン決定
  - ローカル / サテライト配信
  - 設定保持
  - UI 表示

## 4. LED ノードの設計方針

- 各ノードは ESP32-S3 と LED ドライバ機能を持つ
- 大容量アセットは将来的に microSD に保持する
- 中央からは軽量コマンドのみ受信し、各ノードが描画を担当する
- FastLED を出力層として採用する
- 電流は理論最大ではなく運用上限で制御する

## 5. 電源方針

### 現在の試験構成

- 5V 系で駆動
- FastLED による電流制限を有効化
- 実装時点の上限目安: 5V / 2500mA

### 完成形構想

- CIO SMARTCOBY Pro SLIM 35W
- USB-C PD Trigger により 12V 固定出力
- 5V / 8A Buck DC-DC へ降圧
- 5V 母線から各ノードと LED へ給電

## 6. ハード面の重要制約

- WS2812 系 LED は電流変動が大きいため、輝度と同時点灯率の制御が必須
- ESP32-S3 の GPIO はストレージ、LED、通信の兼ね合いで逼迫しやすい
- microSD を 4-bit SDMMC で使う場合は配線難度が上がる
- 将来の量産寄り構成ではノイズ、GND、レベルシフト、コネクタ設計が重要

## 7. まだ固め切っていない項目

- LED ノードの microSD 実装方式
- 1 ノードあたりの最終 LED 数と並列ライン数
- 完成形電源構成での実負荷試験値
- 保護回路とコネクタの量産仕様

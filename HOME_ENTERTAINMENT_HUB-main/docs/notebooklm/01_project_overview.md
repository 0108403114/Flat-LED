# HOME_ENTERTAINMENT_HUB プロジェクト概要

最終更新: 2026-05-09

## 1. 何を作っているか

HOME_ENTERTAINMENT_HUB は、家庭のリビング空間向けの高品位ライティングと、スタジアム規模の大規模同期演出を同じ設計思想で扱う分散型ライティングシステムです。

現在の MVP では、ESP32-S3 ベースのサテライト LED ノード 2 台を使い、以下を実証しています。

- UDP マルチキャストによる同時制御
- 背景アニメーションと歌詞テキストのレイヤー合成
- ノード単位のターゲティング
- シーン切替と遅延実行
- 長時間連続動作

## 2. 設計思想

大規模同期を成立させるため、中央から全フレームを送る方式ではなく、各ノードが自律描画する方式を採用しています。

- データ分散: 重いアセットやフォントは各ノードに持たせる
- 命令一括: 中央からはアセット ID、表示モード、切替時刻などの軽量コマンドだけ送る
- 自律描画: 各ノードの ESP32-S3 が背景とテキストを合成して LED に出力する
- 段階拡張: 2 台 MVP から始め、将来的に多ノード展開へ広げる

## 3. 現在のシステム構成

### 中央側

- TouchDesigner / Python テストツールが制御送信元
- TouchDesigner 側は Execute DAT ベースで controller 自動初期化（再起動後の復元）に対応
- 将来は HomeLiveHall コントローラが中央制御を担う
- 通信はサテライト向けに Wi-Fi UDP マルチキャストを使用

### サテライト LED ノード

- MCU: Waveshare ESP32-S3-Zero
- 台数: 2 台
- ノード識別:
  - Zone 1 / Node 1
  - Zone 1 / Node 2
- LED 構成: 各ノード 88 x 5、2 台合計で 88 x 10
- LED 数: 各ノード 440 LED、合計 880 LED
- 駆動ライブラリ: FastLED

## 5. 30分安定性試験の結果

2026-05-06 に 2 台構成で 30 分連続試験を実施し、正常完了を確認しました。

- 実行内容:
  - 400ms 間隔で CTRL を継続送信
  - 1 分ごとに歌詞表示を更新
  - 2 ノード同時運用を継続
- 結果:
  - M01/30 STAB から M30/30 STAB まで欠落なし
  - exit code 0 で正常終了
  - 終了時に両ノードへ OFF を送信
- 解釈:
  - 現行 MVP 条件では通信継続性と表示継続性は十分安定
  - 追加で RAM 推移や Wi-Fi 再接続回数を観測する場合はシリアルログ試験が必要

## 6. 今どこまでできていて、何がまだか

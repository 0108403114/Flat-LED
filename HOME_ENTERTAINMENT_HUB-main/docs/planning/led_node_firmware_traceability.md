# LEDノード書き込みコード追跡台帳

最終更新: 2026-05-24
目的: 「どのLEDノードに、どのコードを書いたか」を後から確実に追跡できるようにする。

## 1. 現時点で分かっていること

現在のワークスペース情報だけでは、実機LEDノードに最後に書き込まれたコードを断定できません。

理由:
- 直近の実行ログは Controller 側ビルド確認が中心
- LEDノード用の書き込み実行ログ (flash/upload) が台帳化されていない

このため、現時点の状態は「未確定」として扱います。

## 2. LEDノード候補コード

1. ローカルLEDノード (UART)
- ソース: `firmware/lighting_node/main/app_main.c`
- 特徴: UARTフレーム受信、HEARTBEAT監視、WS2812制御

2. サテライトLEDノード (Wi-Fi/UDP)
- ソース: `firmware/satellite_node/src/main.cpp`
- 特徴: UDP受信、SAT_V3制御、FastLED描画

## 3. 今後の記録ルール

LEDノードへ書き込みを行った直後に、以下を必ず記録します。

- 実施日時
- 対象ノード種別 (lighting_node / satellite_node)
- 対象デバイス識別子 (COM、Node ID、ラベル)
- 書き込み元ソースパス
- 書き込み時のGitコミットID
- 実行コマンド
- 結果 (成功/失敗)

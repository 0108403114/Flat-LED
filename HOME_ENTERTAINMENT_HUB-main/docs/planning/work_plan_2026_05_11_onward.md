## ユースケース単位の評価ループ

1. ユースケースを 1 つ選ぶ（LIVE / READY / ENTRY など）
2. シーケンスを実機実行して課題を記録
3. プロトコル改善が必要か判定
4. 必要なら最小変更で実装
5. 再テストして差分確認

評価観点:
- 可読性（背景上の文字視認性）
- 同期性（曲タイミングとのズレ）
- 安定性（ドロップ・フリーズ・セーフティ遷移）
- 後方互換性（旧ノード/旧送信ツール動作）

## プロトコル改善方針（後方互換重視）

- 原則: 既存 CTRL/CONTENT/HEARTBEAT を壊さない
- 拡張案: 追加機能は新パケット（例: 0x50 ADVANCED_CTRL）で分離
- 既存ノードは旧仕様で動作継続、新ノードのみ拡張機能を有効化
- 重要変更は必ず「仕様更新 → 実装 → テスト」の順で反映

# 企画案: HOME_ENTERTAINMENT_HUB

> **作成日**: 2026-05-02  
> **ステータス**: 企画確定・開発中

---

## 1. プロジェクト概要

### プロジェクト名
HOME_ENTERTAINMENT_HUB（ホーム・エンターテインメント・ハブ）

### コンセプト
**「部屋そのものをライブ空間に変える、卓上サイズのインテリジェント照明コントローラー」**

音楽・映像に連動した LED 照明演出と環境センシングを小型デバイスで統合し、ライブ感・没入感のある自宅空間を誰でも手軽に構築できるシステムを目指す。

---

## 2. 背景と課題

### 背景

- 自宅での音楽鑑賞・映像視聴体験の高度化ニーズが高まっている。
- プロ仕様のステージ照明制御は高コスト・大型でホームユース不向き。
- ESP32 + WS2812B の組み合わせは低コストで高い表現力を持つが、音楽連動やマルチノード同期には体系化されたシステムが必要になる。

### 解決する課題

| 課題 | 本プロジェクトの解決策 |
|---|---|
| 照明演出の自動化が難しい | マイク入力＋アセット再生で音楽連動を自動化 |
| 複数の LED を手動で制御する手間 | 中央ノードからの一元コマンド配信 |
| 商用製品では設定の自由度が低い | OSS ファームウェアで全パラメータをカスタマイズ可能 |
| バッテリー駆動で持ち運べない | モバイルバッテリー（CIO SMARTCOBY 35W）で完全コードレス |

## 3. システム構成

### ノード構成図

```

│          HOME_ENTERTAINMENT_HUB          │

│  [ESP32-S3-Zero]                        │
│    ├─ TFT 2.0" (状態表示)                │
│    ├─ マイク INMP441 (音量検出)           │
│    ├─ 照度センサ BH1750                  │
│    ├─ 電流センサ INA228 ×2               │
│    ├─ ボタン SW1〜3 (操作)               │
│    └─ WS2812B イルミ 24LED (外装)        │
│                                         │
│  電源: CIO SMARTCOBY Pro SLIM 35W        │
│    → PD トリガ 12V → 5V/8A Buck DC-DC   │
└──────────────┬──────────────────────────┘
               │ UART（有線直結）
               ▼
┌─────────────────────────────────────────┐
│          LED ノード（ローカル）             │
│  [ESP32-S3-Zero]                        │
│    ├─ WS2812B マトリクス 880LED           │

               │ Wi-Fi（将来対応）
               ▼
┌──────────────────────────┐
│  サテライトノード（複数台）  │
│  Wi-Fi 配信受信           │
└──────────────────────────┘
```

## 4. 主要機能

### 4-1. 中央ノード機能

| 機能 | 説明 |
|---|---|
| **環境センシング** | 照度（BH1750）・音量（INMP441）をリアルタイム計測 |
| **電力監視** | INA228 で 5V 母線の電圧・電流・電力を常時モニタリング |
| **TFT 状態表示** | Home / Power / Sensor の 3 画面をボタンで切替 |
| **ボタン操作** | SW1=モード切替、SW2=シーン切替、SW3=表示/ミュート |
| **LED 演出配信** | UART コマンドで LED ノードに演出を配信 |
| **コードレス運用** | CIO SMARTCOBY 35W で最大 3〜4 時間のバッテリー駆動 |

### 4-2. LED ノード機能

| 機能 | 説明 |
|---|---|
| **LED マトリクス制御** | WS2812B 880LED（10×88）を最大 4 ライン並列出力 |
| **アセット再生** | microSD から .led ファイルを読み出してフレーム再生 |
| **コマンド受信** | UART（ローカル）/ Wi-Fi（サテライト）でコマンド受信 |
| **安全機構** | HEARTBEAT タイムアウトでセーフカラーに自動遷移 |

### 4-3. 演出シーン（予定）

| シーン名 | 概要 |
|---|---|
| **LIVE** | 音量に連動した高速フラッシュ・カラーパルス |
| **HEALING** | 低輝度・スロートランジションのアンビエント演出 |
| **SYNC** | 外部 AudioLINKCORE 信号に同期した演出 |
| **STATIC** | 単色・グラデーション等の固定表示 |

## 5. ハードウェア仕様

| 項目 | 仕様 |
|---|---|
| メイン MCU | ESP32-S3-Zero (Waveshare) — 4MB Flash + 2MB PSRAM |
| LED ノード MCU | ESP32-S3-Zero × 1（将来: XIAO ESP32S3 Plus へ移行候補） |
| ディスプレイ | GMT020-01 2.0" TFT (ST7789VW, 240×320, SPI) |
| マイク | INMP441 (I2S, MEMS) |
| 照度センサ | BH1750 (I2C, 1〜65535 lx) |
| 電力センサ | INA228 ×2 (I2C, 20-bit ADC, 最大 85V) |
| LED | WS2812B 5V 60LED/m |
| LED 総数 | 24（イルミ）+ 880（マトリクス）= 904 個 |
| 主電源 | CIO SMARTCOBY Pro SLIM 35W（10000mAh, 16mm 薄型） |
| 電源変換 | USB-C PD → 12V → 5V/8A Buck DC-DC |
| 通信 | UART（ノード間）、Wi-Fi（サテライト） |

---

## 6. 開発フェーズと完成定義

### MVP（最小検証版）
- 中央ノード + LED ノード 1 台での UART 通信統合
- センシング・表示・LED 演出が同時動作する
- USB-C 5V 直給電で 30 分以上安定動作

### 完成版
- バッテリー駆動（CIO SMARTCOBY 35W）でコードレス動作
- 演出シーン 2 種類以上が切替可能
- AudioLINKCORE 外部制御対応
- 4 時間以上の安定連続動作

## 8. 成功指標

| 指標 | 基準 |
|---|---|
| 安定動作時間 | 4 時間以上の無停止連続動作 |
| LED 応答遅延 | コマンド送信から LED 反応まで < 50ms |
| 電力効率 | バッテリー 10000mAh で 3 時間以上の連続駆動 |
| 演出品質 | 30fps 以上でフレーム落ちなし |
| 可搬性 | バッテリー込みで 1 kg 以下 |

## 9. 将来展望

- **Wi-Fi 多点展開**: サテライトノードを複数台設置し、部屋全体を一元管理
- **HDMI 映像連動**: 映像の平均輝度・色温度に連動した Ambilight 的演出
- **クラウド連携**: スマートホームシステム（Matter / Home Assistant）との統合
- **商品化検討**: MVP 完成後に小規模ロット製造の可能性を評価
